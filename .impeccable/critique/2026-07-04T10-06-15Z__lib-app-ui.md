## Priority Issues

- **[P1] False autodiscovery promise** — `what_is_rss_page` method 2 + `addFeedSubtitle` tell users to paste a plain website address ("verge.com"); `FeedService.fetchFeed` only parses direct RSS/Atom URLs, so following the app's own instructions produces an error.
- **[P1] Dead "Rate the App" button** — settings_screen.dart:638 `onTap: () {}`.
- **[P1] Unlocalized, load-bearing gesture hint** — bookmarks_screen.dart:43 hardcoded English "Swipe left on any article to save it here."; the swipe-right (mark read) gesture has zero discovery anywhere.
- **[P1] Onboarding dead-end** — Skip only renders when both feed lists fail; error state has no Retry button though copy says "try again".
- **[P1] Interval display lie** — settings_screen.dart:437 coerces unknown values to "30 Seconds" while the actual default is 1800 s.
- **[P1/P2] Technical error leak** — errorAddingFeed shows "Could not fetch or parse feed." English-only inside Turkish UI, no actionable advice.
- **[P2] Hidden feature cluster** — per-feed keyword filter, full-text default, and notification mute only reachable via long-press on drawer feed rows; no hint exists.
- **[P2] Tap targets below 48 dp** — drawer `more_vert` (~26 px, no tooltip/Semantics), thumbnail bookmark chip 24 px.
- **[P2] Contrast** — 11–13 px labels at 50–55 % onSurface alpha ≈ 3.4:1, below WCAG AA 4.5:1 (feed card date, drawer counts, section headers).
- **[P2] Empty state without action** — "No feeds match your current filter." offers no clear-filter/show-all action.
- **[P2] Contradictory sync copy** — "Sync Background … while app is open" vs "Auto Refresh … in background".
- **[P3] Version hardcoded** — drawer footer + about dialog + licenses page say 1.0.0; app is 1.0.2.
- **[P3] Unread filter icon** — visibility/visibility_off mapping ambiguous (tooltip mitigates).

## Persona Red Flags

**Jordan (First-Timer)**: follows in-app RSS guide, pastes "verge.com", gets a semi-technical English error. Never discovers swipe-to-save or long-press feed options. Cannot skip onboarding.

**Casey (Distracted Mobile)**: drawer three-dot target ~26 px misses; onboarding on flaky connection dead-ends with no retry.

**Meral (project persona: Turkish, non-technical)**: sees English strings in bookmarks empty state and in every feed-add failure; "30 Seconds" label vs 30-minute reality erodes trust.

## What's Working

- Onboarding is 2 taps to a working feed list — genuinely excellent activation.
- "What is RSS?" page is rare, well-written contextual help.
- Empty home state offers a direct "Add feed" action; offline banner is clear; skeleton loading is correct product-register behavior.
- Settings grouping is logical; destructive actions (factory reset, delete) all confirm.
